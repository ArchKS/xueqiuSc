# 项目开发变更日志 (log.md)

## 2026-03-12

### 1. 项目初始化
- **操作**：创建了 Python 虚拟环境，并初始化了 `requests` 和 `pandas` 依赖。
- **目标**：搭建基础爬虫运行环境。

### 2. 反爬攻坚 (从 Requests 切换到 DrissionPage)
- **挑战**：雪球网使用了阿里云 WAF 防火墙及 `debugger` 动态反调试，导致 `requests` 请求被拦截（返回 403 或 JS 挑战页）。
- **优化**：改用 `DrissionPage` 驱动 Chromium 浏览器内核。
- **成果**：成功通过 WAF 验证，并能正常执行混淆的 JS 签名（如 `md5__1038`），不再需要手动计算动态参数。

### 3. 正文全量抓取功能
- **优化**：针对列表页只显示摘要的问题，开发了详情页自动跳转抓取逻辑。
- **清洗**：新增 `clean_html` 方法，使用正则剔除 `<a>` 等 HTML 标签，保留纯文本。

### 4. 智能动态频率控制 (Anti-Ban)
- **优化**：实现了“加载耗时扣除”算法。
- **公式**：`实际休息时间 = 目标停留(1s + 字数/1000) - 已加载耗时`。
- **成果**：在保证账号安全的前提下，极大提升了对短篇内容的抓取速度。

### 5. 任务进度与时间预估 (UX 增强)
- **优化**：在控制台输出中增加了已用时间、预计剩余时间（ETA）和总体进度统计。
- **功能**：自动计算平均耗时，实时预估全量任务完成点。

### 6. 全量数据采集优化
- **优化**：通过第一页 API 返回的 `maxPage` 和 `total` 字段，实现自动全量抓取，无需手动输入页数。
- **演进**：
    - **初步**：使用 `...` 字符串匹配摘要末尾判断是否截断。
    - **终极**：利用 API 原生 `truncated` 布尔值字段进行 100% 精准判断，彻底避免了对短内容的重复跳转，效率提升 3-5 倍。

### 7. 异常处理与编码修复
- **优化**：在 `analyze_user.py` 中添加了多重编码（`utf-8-sig` -> `gb18030`）尝试机制。
- **成果**：解决了 Windows 环境下处理包含中文字符的 CSV 文件时出现的 `UnicodeDecodeError`。

### 8. 项目结构整合 (CLI)
- **新增**：创建了 `main.py` 主入口，支持命令行参数：`python main.py <UID> [max_pages]`。
- **文档**：编写了 `Usage.md` 和本开发日志 `log.md`。
